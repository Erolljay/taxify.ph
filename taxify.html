Txform Now! helps this business generate BIR-compliant tax returns and reports using data already in Manager.io — no double-entry needed.

Recommended setup order for a new business:

  1. Business — BIR identity fields (TIN, RDO Code, classification, registered name, address).
  2. Chart of Accounts — create/rename GL accounts and groups, and map each one to a BIR income-tax category (Asset, Liabilities, Equity, Revenue, Cost of Sales, Cost of Services, Operating Expenses, Other Income, Other Expense) so 1701/1701Q/1702Q/1702RT classify them correctly.
  3. Tax codes — create standard PH tax codes, then map VAT, EWT / CWT, FWT, and Percentage Tax categories — including the GL account each VAT code should post to.
  4. Tax Rates — review the VAT/Percentage Tax rate, the individual graduated income tax table, and corporate/MCIT rates that returns are computed with. Add a new dated entry whenever BIR introduces a rate change — past returns keep using whatever rate was in force at the time.
  5. Setup — install Batch Import tools for Sales, Purchases, Payroll, Receivables, and Payables.
  6. Payslip items — assign payslip items to BIR reporting categories, plus the expense / liability account each one posts to, so values flow into 1601C, SAWT, and 2316 correctly.
  7. Reports — install the BIR report buttons this business needs.

Customer / Supplier / Employee BIR details (TIN, addresses, SSS / PhilHealth / Pag-IBIG numbers, etc.) are filled in directly within the relevant report screens, not here.

Once setup is done, go back to Data Intake in the sidebar to start preparing filings.

Install a BIR report to make it available under this business's Reports tab.

Per-row buttons create one tax code at a time.

Install Batch Import buttons. Once installed, each one appears directly on the Sales Invoices, Purchase Invoices, Receipts, or Payments page, letting the bookkeeper upload a simple Excel sheet that gets posted straight into Manager.
