🇵🇭 Philippines BIR

This extension helps Philippine businesses generate BIR-compliant tax returns and reports using data already in Manager.io — no double-entry needed.

Recommended setup order for a new business:

  1. Business — BIR identity fields (TIN, RDO Code, classification, registered name, address).
  2. Chart of Accounts — create/rename GL accounts and groups, and map each one to a BIR income-tax category (Asset, Liabilities, Equity, Revenue, Cost of Sales, Cost of Services, Operating Expenses, Other Income, Other Expense) so 1701/1701Q/1702Q/1702RT classify them correctly.
  3. Tax codes — create standard PH tax codes, then map VAT, EWT / CWT, FWT, and Percentage Tax categories — including the GL account each VAT code should post to.
  4. Setup — install Batch Import tools for Sales, Purchases, and Payroll.
  5. Payslip items — assign payslip items to BIR reporting categories, plus the expense / liability account each one posts to, so values flow into 1601C, SAWT, and 2316 correctly.
  6. Reports — install the BIR report buttons this business needs.

Customer / Supplier / Employee BIR details (TIN, addresses, SSS / PhilHealth / Pag-IBIG numbers, etc.) are filled in directly within the relevant report screens, not here.

Select a business above, then switch to any tab to get started.

Install a BIR report for the selected business to make it available under that business's Reports tab.

Per-row buttons create one tax code at a time in the selected business.

Install Batch Import buttons for the selected business. Once installed, each one appears directly on that business's Sales Invoices or Purchase Invoices page, letting the bookkeeper upload a simple Excel sheet that gets posted straight into Manager.
